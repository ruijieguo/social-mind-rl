### 1.2 诊断

**3 大瓶颈**:
1. **Hi-ToM direct (P1)**: 直接模式禁 CoT，model 必须"心算" 5-阶 belief tracking。DeepSeek 反正通过 reasoning_content 偷跑 CoT 所以拿到 0.80+；我们 Qwen 直出 token=64 几乎不可能正确回答 order_2+。**解决**: 训练时给 direct 协议数据，让 model 学会"压缩 CoT 到 forward pass"。
2. **EmoBench EU_emotion (P1)**: 6-选项细粒度情绪命名 (Happiness/Sadness/Anger/Fear/Surprise/Disgust + 12 derived)。500 条合成 → 0.50 acc。需要 ~2000+ 条高质量分布数据。
3. **ToMBench Belief (P2)**: 加权数据集已有 2236 条，但 v3.1 0.7359 → v3.2 0.7359 持平，**说明数据质量而非数量是瓶颈**。需要 DeepSeek/GPT-5.5 风格 distillation 教 Belief 推理模式。

**良性观察**:
- ToMBench 已有 3 个格子超 DeepSeek (Non-literal Comm direct, Knowledge del_tom, False Belief del_tom)
- Hi-ToM cot/del_tom 训练超有效，v3.2 0.7150/0.7550 已接近 DeepSeek cot 0.7475
- SocialIQA gap 已经很小 (+2.2~3.5pp)，少量增量数据可能就能闭合

### 2.2 关键创新: Hi-ToM "direct-style compressed CoT"

**Hypothesis**: DeepSeek direct 0.80 是因为 reasoning model 通过隐藏 reasoning_content 偷跑 CoT。Qwen3 direct 没这条路。但 model 可在训练中学到**单次 forward pass 完成 belief tracking**（类似 transformer 的 in-context 计算）。

**Recipe**:
```python
# Hi-ToM 1200 条 → 派生 1000 条 direct-style
for r in hitom_train_1200:
    if r['task'] in ['order_1', 'order_2', 'order_3', 'order_4']:
        # Pair: cot version (existing) + direct version (new)
        direct_r = {
            **r,
            "messages": [
                {"role": "system", "content": "<direct system prompt: no reasoning, output \\boxed{X} only>"},
                r['messages'][1]  # user same
            ],
            "question_id": f"{r['question_id']}_direct"
        }
        yield direct_r
```

reward worker 已支持 task-aware l_max — direct 数据走 l_max=64 (短)，cot/del_tom 走 l_max_long=512。

### 2.3 数据 dedup 与质量门

- MinHash 4-gram Jaccard ≤0.6 vs 4 个 eval 集 (已有 pipeline)
- 新数据先在 v3.2 ckpt-270 上跑 dry-run，丢弃 pass_rate >0.95 (过简) 和 ≤0.05 (过难，模型无法学) 的样本
- 保留 pass_rate ∈ [0.10, 0.85] 的"学习甜区"

### 3.1 Reward worker 改动

继续基于 v3.2 的 A-Z 支持 + task-aware l_max。

**新增需求**: direct 协议数据强制短输出 (l_max=64) — 但要避免奖励 boxed-without-reasoning 模式而损害 cot 数据。方案: source 检测 — `source ∈ {hitom_synth_direct, ...}` 的 direct-style 数据，用 `l_max=64`；其余按当前逻辑。

```python
if 'direct' in source.lower():
    l_max_eff = 64.0  # force compressed answers
elif needs_long:
    l_max_eff = self.l_max_long
else:
    l_max_eff = self.l_max
```

## 5. 风险与备选方案

### 风险 1: Hi-ToM direct 即使加数据仍不达 0.80
- **风险**: model 容量限制 — 14B 在 64-token forward pass 内完成 5-阶 belief tracking 可能本质不可行
- **应对 Plan B**: 让 direct 协议**也允许短 CoT** (sys prompt: "think briefly then answer in \\boxed{X}", max 100 tokens) — 这与 DeepSeek 通过 reasoning_content 偷跑等价。重新评测但用同一 protocol 描述。

### 风险 2: 新数据过多稀释 ToMBench backbone
- **风险**: v3.2 已观察到 ToMBench -0.3~0.5pp。v3.3 新加 4500 条 → ToMBench 比例从 91% 降到 76%
- **应对**: ToMBench Belief distillation 600 条**补强**ToMBench 而非稀释；其它新数据走 task-weighted sampling，让 ToMBench backbone 仍占 ~75%

### 风险 3: EmoBench EU_emotion 1500 条仍不够
- **风险**: 6-opt 细粒度情绪本质难 (DeepSeek 也只 0.70)
- **应对**: 加 DeepSeek-style "explicit reasoning + boxed answer" demonstrations，从 distillation 角度而非 pure synth

### 风险 4: 续训 v3.2 直接退化 (类似 Stage 13)
- **风险**: v3.2 已收敛，更小 LR 续训可能微动
- **应对**: 与 v3.2 不同的数据分布 (+4500 新数据, +25% 大小) 强制让梯度流向新分布；step 20 val 退化 ≥1pp 即停
