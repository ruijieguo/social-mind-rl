# v3.5 Training Plan — Improved GPT-5.5 Distillation

> **Date**: 2026-05-26
> **Predecessor**: v3.4 (Stage 18 ckpt-30, GPT-5.5 paraphrase + verify distillation)
> **Hypothesis**: v3.4 蒸馏在 EmoBench/SocialIQA 上失效是因为方法太"轻量" —— 仅用 letter-level paraphrase + verify。需要更深度的蒸馏：完整 reasoning trace SFT + 知识结构注入 + 多采样投票。

---

## 1. v3.4 失败模式诊断（基于错例对比）

### 1.1 错例三分类

| Bench × Proto | v34 错+DS 对 (recoverable) | 两者都错 (knowledge gap) | v34 对+DS 错 (label noise) |
|---|---|---|---|
| **EmoBench EU_emotion direct** (n=400) | **105** (26.3%) | **102** (25.5%) | 18 (4.5%) |
| SocialIQA direct (n=1954) | 182 (9.3%) | 235 (12.0%) | 136 (7.0%) |
| SocialIQA cot | 186 | 249 | 121 |
| SocialIQA del_tom | 164 | 254 | 100 |

**关键洞察**:

1. **EmoBench EU_emotion**: 26% 是"教得会"的 (DS 会做 + v34 不会)，25% 是 base 知识盲区 (两者都错) — 这是 6-option 细粒度情绪命名，base 14B 的世界知识就有限制
2. **SocialIQA**: 才 9-10% 是真正可教，bothW 12% 不可教，**v34R+DSW 7%** 是 label noise (SocialIQA 题目本身就有歧义) — distillation 上限就是 ~10pp，**接近 v3.4 实际表现 0pp**，说明蒸馏完全失败
3. **SocialIQA del_tom 比 cot 多 100-136 个 v34R+DSW**: 投票降噪也救不了，说明这些题是 base model 早就答对了，不是"训练让它学会的"

### 1.2 v3.4 蒸馏方法论问题

| 问题 | 表现 | 修复方向 |
|---|---|---|
| 仅蒸馏字母答案 | model 学不到推理路径 | **CoT-trace SFT** 注入 GPT-5.5 完整推理 |
| 单次 GPT-5.5 verify (T=0) | 引入 noise (GPT-5.5 也会偶错) | **3-5 sample voting**, ≥2 一致才保留 |
| 与原始 RL 数据按比例混合 | 蒸馏数据被稀释 | **SFT pretrain phase**, 蒸馏数据先单独 SFT 1 epoch |
| 没考虑 base 知识盲区 | 即使 GPT-5.5 答对，14B 也学不会 | **知识结构注入**: 显式给 emotion ontology / 文化常识 |
| Reward 仍只看字母对错 | distillation chain 被截断 | **Process reward** (中间步骤监督) |

## 2. v3.5 设计：四管齐下

### 2.1 改造 1 — 多采样投票蒸馏（**Sample-Voted Distillation**）

**当前 (v3.4)**:
```
Q → GPT-5.5(T=0) → A_gpt
if A_gpt == gold: keep paraphrase, label = gold
```

**v3.5**:
```
Q → GPT-5.5(T=0.3) × 5 samples → [A1, A2, A3, A4, A5]
votes = Counter([extract(a) for a in samples])
top_letter, top_count = votes.most_common(1)[0]
if top_letter == gold and top_count >= 4:  # 5 中至少 4 个一致
    keep one trace where extract == gold
    label = (gold, full_trace)
elif top_letter == gold and top_count == 3:
    drop  # confidence not high enough
```

**效果预估**:
- 当前 keep rate ~50% → v3.5 ~30% (更严格)
- 但保留的样本质量更高，noise rate < 5%
- 直接缓解 SocialIQA 7% label noise 问题

### 2.2 改造 2 — CoT-trace SFT Pretrain Phase

**当前 (v3.4)**:
- 蒸馏数据 + 原 ToMBench RL 数据 直接 GRPO 训练
- 只见字母 reward，不见 GPT-5.5 reasoning

**v3.5 新增 SFT 阶段**:

```
Stage 19a (SFT): 1 epoch, lr=1e-5, AdamW
  - 输入: distill_traces.jsonl (Q + GPT-5.5 full <think>...</think>...\boxed{X})
  - loss: full token CE on assistant tokens
  - 目的: 教 14B 模仿 GPT-5.5 的推理结构

Stage 19b (RL): GRPO on TomBench/Hi-ToM/EmoBench/SocialIQA mixed
  - 用 19a 后的模型作为 init
  - reward: 字母正确性 + length penalty
  - 目的: 让 SFT 学到的 reasoning 在 reward 信号下精炼
```

**关键技术细节**:
- SFT phase 用 `transformers` Trainer (不是 Megatron) 跑 1 epoch on 1×8 H800，~2h
- 转 Megatron 后跑 RL phase
- 损失权重: SFT phase 把蒸馏 trace 中字母 token 加权 5×（强调最终答案）

### 2.3 改造 3 — 知识结构注入（**Knowledge-Augmented Distillation**）

针对 EmoBench EU_emotion 的 102 bothW 题 + 105 recoverable 题，问题不仅是"推理"，是"对情绪概念的细分"。

**v3.5 步骤**:

1. **从 GPT-5.5 提取 emotion ontology** (一次性):
   ```
   Prompt: "List 12 fine-grained emotions in Chinese contexts that frequently appear in EmoBench EU_emotion. For each: definition, typical situational triggers, distinguishing feature from similar emotions."
   ```
   产出 `data/distill/emotion_ontology.jsonl` (~12 entries)

2. **将 ontology 注入训练 prompt** (可选 system message):
   ```
   System: "You are answering an emotion identification question. Note that fine-grained emotions can be: 内疚 (guilt — about past actions), 羞愧 (shame — about self-identity), 委屈 (grievance — feeling treated unfairly), 沮丧 (frustration — at obstacles), ..."
   ```

3. **专项错例 SFT** (~200 条):
   - 取 EmoBench EU_emotion 中 v3.4 错 + GPT-5.5 对 的 105 题
   - 用 GPT-5.5 生成完整 reasoning + 对比错误候选 ("不是 X 因为...而是 Y 因为...")
   - SFT 加入 stage 19a

### 2.4 改造 4 — Process Reward (中间步骤监督)

**当前**: reward = (final letter == gold) ? 1 : 0
**v3.5**: 引入 reasoning 质量分量

```python
def compute_reward(response, gold):
    letter_pred = extract_boxed(response)
    letter_correct = (letter_pred == gold)

    # 新增: reasoning 结构分
    reasoning_text = extract_reasoning(response)  # <think>...</think>
    has_belief_track = bool(re.search(r'(believes|thinks|knows)\s+(that|about)', reasoning_text))
    has_alt_consideration = bool(re.search(r'(however|but|alternatively|on the other hand)', reasoning_text))
    structure_bonus = 0.1 * (has_belief_track + has_alt_consideration)

    if letter_correct:
        return 1.0 + structure_bonus
    else:
        # 答案错但有合理 reasoning，给 0.1 部分分
        return 0.1 * (has_belief_track + has_alt_consideration) / 2
```

**优势**: 避免 model 学会"瞎猜字母"作弊，强制 reasoning 表述。Hi-ToM 上特别有效（5-order belief 必须显式追踪）。

## 6. Failure modes & rollback

1. **5-sample voting retain rate 太低** (<20%) → 降到 ≥3/5 一致即可
2. **SFT 后模型 collapse** → 减小 lr 到 5e-6 或减少 epoch
3. **Process reward 引入 hacking** (model 输出 boilerplate "however... but...") → 增加 length penalty
4. **如果 v3.5 不超 v3.4**: 回滚到 v3.4 production_frozen, 接受 Hi-ToM cot -0.42pp 是当前 14B 的天花板
